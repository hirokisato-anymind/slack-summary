# Slackタスクサマリ生成システム

このプロジェクトは、Slackの会話履歴を分析し、未完了・進行中のタスクを抽出してサマリを生成するClaude Code支援ツールです。

## プロジェクト構成

```
slack-summary/
├── CLAUDE.md                        # このファイル（Claude Codeへの基本指示）
├── context/
│   └── channels/
│       ├── _template.md             # チャンネル設定テンプレート
│       └── {チャンネル名}.md         # 各チャンネルの設定・過去サマリ
└── .claude/
    └── commands/
        └── task-summary.md          # /task-summary コマンド定義
```

## 基本的な使い方

- `/task-summary {チャンネル名}` — 指定チャンネルの未完了タスクサマリを生成する
- `/task-summary {チャンネル名} {期間}` — 期間を指定して生成（例: `7d`, `2w`, `1m`）
- チャンネル設定は `context/channels/{チャンネル名}.md` で管理する

## Claude Codeへの行動指針

### タスク抽出の基準

以下のパターンをタスクとして認識する:
- 明示的な依頼・お願い（「〜してください」「〜お願いします」「〜よろしく」）
- TODO・やること宣言（「〜やります」「〜対応します」「〜確認します」）
- 質問・確認待ち（回答がないまま放置されているもの）
- レビュー依頼・承認待ち
- 期限付きのアクション（「〜までに」「今週中に」）
- リマインド・フォローアップ依頼

### タスクのステータス判定

Slackのスレッド・リアクション・後続メッセージを分析し、以下のステータスを判定する:
- **完了**: 完了報告あり、✅リアクション、「done」「対応しました」等の返信
- **進行中**: 着手の報告があるが完了報告がない
- **未着手**: 依頼後に進捗の言及がない
- **確認待ち**: 質問や確認に対して返答がない

### 分析の原則

- 事実（実際のメッセージ内容）に基づいて判定する
- 曖昧な場合は「未確認」として扱い、推測で完了と判断しない
- タスクの担当者を可能な限り特定する
- 期限が明示されているものは優先的に表示する
- 重複するタスク（同じ件について複数回言及）は1つにまとめる

### やってはいけないこと

- 雑談や日常会話をタスクとして拾わない
- 完了が明確なものを残タスクとして報告しない
- メッセージの文脈を無視して断片的にタスク化しない
- 個人のプライベートな内容に言及しない
