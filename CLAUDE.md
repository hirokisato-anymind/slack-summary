# 1on1マネジメント支援システム

このプロジェクトは、メンバーとの1on1を起点にした成長管理をClaude Codeで支援するためのものです。

## プロジェクト構成

```
1on1-management/
├── CLAUDE.md                    # このファイル（Claude Codeへの基本指示）
├── context/
│   └── members/                 # メンバーごとの成長追跡ファイル
│       ├── _template.md         # メンバーファイルのテンプレート
│       └── {名前}.md            # 各メンバーの成長追跡ファイル
└── .claude/
    └── commands/
        └── 1on1-prep.md         # /1on1-prep スキル定義
```

## 基本的な使い方

- `/1on1-prep {名前}` — 指定メンバーの1on1準備資料を自動生成する
- メンバーファイルは `context/members/{名前}.md` に蓄積されていく
- 1on1後の気づきや観察はメンバーファイルに追記して育てていく

## Claude Codeへの行動指針

### メンバー情報の扱い
- `context/members/` 配下のファイルは機密情報として扱う
- メンバーの情報を読み込む際は必ず最新のファイルを参照する
- 1on1後の気づきを記録する際は、日付を付けて追記する

### 評価の観点
- 事実（Slackの発言、MTG議事録、成果物）に基づいて分析する
- 感情的・主観的な評価は避け、観察できた行動を根拠として示す
- 成長の変化を時系列で把握し、短期的な波に左右されない視点を持つ
- ポジティブな行動（自走、後進育成、主体性）は具体的に拾い上げる

### 1on1準備の心構え
- アクションアイテムは一つ残らず追跡する
- 前回から今回の変化・成長を必ず言語化する
- 褒めるポイントは「いつ・何を・なぜ良い」の形式で具体的に提示する
- 目標ロードマップに対する現在地を明示する

### やってはいけないこと
- AIの出力をそのままメンバーに読み上げるような使い方を促さない
- データのない主観的な評価を断定的に述べない
- メンバーの個人情報をコンテキスト外に出力しない
