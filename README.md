# Slack タスクサマリ生成ツール

Slackの会話履歴をClaude Codeで分析し、チャンネル内の未完了タスク・確認待ち事項を自動でサマリ化するツールです。

## クイックスタート

### 1. チャンネル設定ファイルを作成する

```
context/channels/_template.md をコピーして
context/channels/{チャンネル名}.md を作成する
```

チャンネルID・監視対象・除外パターンなどを記入する。

### 2. タスクサマリを生成する

```
/task-summary {チャンネル名}
```

Claude Codeが自動で以下を実施する:
- 指定チャンネルの直近2週間のメッセージを取得
- タスク・依頼・確認待ち事項を抽出
- スレッド・リアクションから完了状況を判定
- 担当者・期限・ステータス別にサマリを生成

### 3. 期間を指定して生成する

```
/task-summary {チャンネル名} 7d    # 直近7日間
/task-summary {チャンネル名} 1m    # 直近1ヶ月
```

## 連携ツール

| ツール | 用途 | 必須/任意 |
|--------|------|---------|
| Slack MCP | メッセージの取得・分析 | 必須 |

---

## ファイル構成

```
slack-summary/
├── CLAUDE.md                        # Claude Codeへの基本指示
├── README.md                        # このファイル
├── context/
│   └── channels/
│       ├── _template.md             # チャンネル設定テンプレート
│       └── {チャンネル名}.md         # 各チャンネルの設定・過去サマリ
└── .claude/
    └── commands/
        └── task-summary.md          # /task-summary コマンド定義
```

---

## 活用のコツ

- **定期的に実行する**: 週次や日次でサマリを生成し、タスクの取りこぼしを防ぐ
- **チャンネル設定を整える**: 除外パターン（bot投稿など）を設定すると精度が上がる
- **過去サマリを蓄積する**: チャンネルファイルに過去のサマリを残すことで、長期間追跡できる
