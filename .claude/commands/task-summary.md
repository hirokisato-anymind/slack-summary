Slackチャンネルの未完了タスクサマリを生成してください。対象: $ARGUMENTS

## あなたがやること

以下のステップを順番に実行し、最後にタスクサマリを出力してください。

### Step 1: チャンネル設定の読み込み

引数を解析してください。
- 第1引数: チャンネル名（必須）
- 第2引数: 期間（任意、デフォルト: `2w`）
  - `7d` = 7日間, `2w` = 2週間, `1m` = 1ヶ月

`context/channels/{チャンネル名}.md` を読み込んでください。
- ファイルが存在しない場合は「チャンネル設定ファイルが見つかりません。`context/channels/_template.md` を参考に作成してください」と伝えて終了する。

### Step 2: Slackメッセージの取得

チャンネル設定ファイルのチャンネルIDを使い、指定期間のメッセージを取得する。

取得時の注意:
- スレッドの返信も含めて取得する
- リアクション情報も取得する
- チャンネル設定の除外パターンに該当するメッセージ（bot投稿など）は除外する

### Step 3: タスクの抽出

メッセージを分析し、以下のパターンに該当するものをタスクとして抽出する:

| パターン | 例 |
|---------|-----|
| 明示的な依頼 | 「〜してください」「〜お願いします」「〜よろしく」 |
| TODO宣言 | 「〜やります」「〜対応します」「〜確認します」 |
| 質問・確認待ち | 回答がないまま放置されている質問 |
| レビュー依頼 | PR/ドキュメントのレビュー依頼 |
| 期限付きアクション | 「〜までに」「今週中に」 |
| フォローアップ | 「リマインドお願い」「後で確認」 |

### Step 4: ステータス判定

各タスクについて、後続のメッセージ・スレッド・リアクションを確認し、ステータスを判定する:

- **完了**: 完了報告あり / ✅リアクション / 「done」「対応しました」「完了」等
- **進行中**: 着手報告あり、完了報告なし
- **未着手**: 依頼後に進捗の言及なし
- **確認待ち**: 質問や確認依頼に返答なし

### Step 5: 整理・重複排除

- 同一タスクへの複数言及を1つにまとめる
- 担当者を特定する（メンション先、返信者など）
- 期限が明示されているものを抽出する
- 緊急度を判定する（期限超過 > 期限間近 > 期限なし）
